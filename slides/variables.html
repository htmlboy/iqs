Variables

And files setup

html {
  --color: hotpink;
}
p {
  color: var(--color);
}

CSS allows us to centralize values (sizes, colorsâ€¦) in variables, which eases design updates. The variables can be declared globally in the html or :root selector (the name of the variables always starts with --) and can be used with the var() function and the name of the variable.

We can declare the variables in a context different from :root, and they will be available in this context. We can also overwrite them on a specific node. 

@import url(general.css);
@import url(typography.css);

The main style file can include external files with the @import rule. This allows to have a single style file linked in the head, and the styles divided into several files to make working with them easier. 

@import url(variables.css);
@import url(general.css);
@import url(typography.css);
@import url(buttons.css);
@import url(layout.css);
@import url(header.css);
@import url(footer.css);
@import url(gallery.css);
@import url(video.css);
@import url(related-products.css);

Divide the CSS into different files according to their function. Arrange them from the most general to the most specific. Centralize variables in one or more files to have more consistent styles.
