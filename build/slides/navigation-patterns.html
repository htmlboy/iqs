Responsive navigation

The main problem with medium and small screens is that navigation, which is a shortcut to access the rest of the web, becomes an awkward element.

The most common solution is to hide the navigation behind an actionable element, although it is important to question the role the navigation is playing and if it is responding adequately to the user's needs.

The simplest pattern is to leave the navigation at the bottom and add an anchor on top.

The toggle pattern creates a drop-down with JavaScript.

The flyout navigation displays the menu from a side.

Priority+ navigation is an interesting way to show at least part of the navigation, instead of hiding everything.

The same Priority+ philosophy can be applied to other navigation formats.

In the case of multi-level navigations, a possible solution is to create nested drop-downs.

But another solution is that subnavigation is not available when there is no space; and the content of every page allows us to navigate (example: WWF).
